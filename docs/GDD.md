# RUGO — Game Design Document

## 1. Обзор игры

| Параметр | Значение |
|----------|----------|
| **Название** | RUGO |
| **Жанр** | 3D Action-Adventure Platformer |
| **Платформа** | Android (первично), Web (прототип) |
| **Стиль** | Лего-фигурки, реалистичный пластик, качественное освещение |
| **Целевая аудитория** | 8—40 лет, любители приключенческих игр |
| **Бизнес-модель** | Free-to-Play + In-App Purchases (покупка героев) |
| **Движок прототипа** | Three.js (WebGL) |
| **Движок продакшена** | Unity (URP) — после утверждения прототипа |

---

## 2. Сеттинг и история

Мир RUGO состоит из **конструкторских блоков** — пластмассовая вселенная, населенная
живыми фигурками. Каждый герой собирается из деталей и имеет уникальные способности.

Тёмный Мастер похищает ** Energy Crystals** — источник жизни мира.
Он раздал кристаллы своим приспешникам — **Антигероям**, каждому на своём уровне.
Игрок проходит уровни, выполняет задания и побеждает Антигероя,
чтобы вернуть кристалл и открыть следующий уровень.

## 3. Основной игровой цикл (Core Loop)

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   ВЫБОР ГЕРОЯ ──> УРОВЕНЬ ──> ЗАДАНИЯ ──> БОСС ──> ПОБЕДА │
│        ^              │           │          │           │
│        │              v           v          v           │
│        │          ИССЛЕДОВАТЬ  СОБИРАТЬ   СРАЖЕНИЕ       │
│        │              КРИСТАЛЛЫ                             │
│        │                                                   │
│        └────────── МАГАЗИН ГЕРОЕВ <── ВАЛЮТА <─────────────┘
│                         │
│                    ПОКУПКА

└─────────────────────────────────────────────────────────┘
```

**Цикл:** Играть уровень → Заработать монеты → Купить/улучшить героя → Пройти сложнее уровень → ...

## 4. Герои (игровые персонажи)

### 4.1. Характеристики

Каждый герой описывается следующими параметрами:

| Параметр | Диапазон | Описание |
|----------|----------|----------|
| `health` | 50—200 | Запас здоровья |
| `speed` | 3—8 | Скорость передвижения |
| `jumpPower` | 5—12 | Сила прыжка |
| `attack` | 10—50 | Урон базовой атаки |
| `attackSpeed` | 0.5—2.0 | Атак в секунду |
| `attackRange` | 1—15 | Дальность атаки (ближний/дальний бой) |
| `defense` | 0—50% | Снижение получаемого урона |
| `special` | — | Уникальная способность (кулдаун 15—30 сек) |

## 5. Антигерои (Боссы)

Каждый уровень заканчивается ареной с боссом. **Нельзя перейти на следующий
уровень, не победив босса.** Каждый босс имеет уникальную механику.

### 5.1. Фазы боя

Все боссы имеют **2—3 фазы** по уровню здоровья:

```
Фаза 1 (100%—66% HP):  Базовые атаки, изучение паттерна
Фаза 2 (66%—33% HP):   Добавляются новые атаки, ускорение
Фаза 3 (33%—0% HP):    Финальный берсерк, максимум опасности
```

```
Здоровье:        500
Атаки:
  - Удар когтём:     ближний, 15 урона
  - Прыжок-топот:    AoE волна, 20 урона
Слабость:         Медленный, уязвим после прыжка (3 сек)
Фазы:             2 (на 50% HP начинает топот чаще)
Награда:          500 монет + Energy Crystal
Обучение:         Знакомство с механикой боя (уклонение + атака)
```

```

  - Огненный шар:     снаряд, 20 урона
  - Взмывание:        взлетает, уходит от ближних атак
Слабость:        Уязвим к дальним атакам в полёте;
                 после дыхания — 4 сек восстановления
Фазы:            3 (на 30% HP — двойной огненный шар)
```

## 6. Дизайн уровней

### 6.1. Структура уровня

Каждый уровень состоит из **3 зон**:

```
[ЗОНА 1: Исследование]  ──>  [ЗОНА 2: Задания]  ──>  [ЗОНА 3: Арена босса]
```

| Зона | Что происходит | Длительность |
|------|----------------|--------------|
| Исследование | Платформинг, сбор монет, ловушки, мини-враги | ~2 мин |
| Задания | Сбор предметов / активация механизмов / defeat N врагов | ~3 мин |
| Арена босса | Бой с антигероем (дверь закрывается, выход только через победу) | ~3 мин |

### 6.2. Типы заданий (Objectives)

```javascript
// Задания, которые разработчик может комбинировать в JSON:
"collect"     // Собрать N предметов (кристаллы, ключи, артефакты)
"defeat"      // Победить N врагов определённого типа
"reach"       // Добраться до точки (checkpoint, портал)
"survive"     // Выжить N секунд (волна врагов)
"protect"     // Защитить объект от уничтожения
"activate"    // Активировать N переключателей (пазлы)
```

### 6.4. Расширяемость уровней

Разработчик добавляет новый уровень **без изменения кода**:

1. Создаёт новую сцену/карту в редакторе
2. Добавляет запись в `configs/levels.json` (см. раздел 9)
3. Опционально: добавляет конфиг босса в `configs/bosses.json`
4. Уровень автоматически появляется в меню выбора

## 7. Боевая система

### 7.1. Управление

#### Клавиатура (PC / прототип):
```
WASD / Стрелки    — движение
Пробел            — прыжок
ЛКМ / J           — базовая атака
ПКМ / K           — специальная способность
Shift             — рывок/уклонение (dash)
E                 — взаимодействие (предметы, рычаги)
Q                 — использовать аптечку
```

#### Тач-управление (Android):
```
Левый джойстик          — движение (виртуальный, появляется по тапу слева)
Правая сторона экрана   — кнопки:
                          [Атака] (большая)
                          [Прыжок]
                          [Способность]
                          [Рывок]
Кнопки полупрозрачные   — не перекрывают графику
```

### 7.2. Механики боя

| Механика | Описание |
|----------|----------|
| **Базовая атака** | Нажать кнопку атаки → анимация удара → урон в радиусе |
| **Комбо** | 3 быстрых нажатия → комбо-удар (×1.5 урона) |
| **Уклонение (Dash)** | Shift + направление → рывок с неуязвимостью 0.5 сек |
| **Способность** | Уникальная атака/защита, кулдаун 15—30 сек |
| **Парирование** | Нажать атаку в момент удара врага → отбрасывание (только Rugo/Titan) |
| **Прыжок-атака** | Атака в воздухе → удар по площади при приземлении |

### 7.3. Получение урона и здоровье

```
Здоровье героя:     Полоса HP (значки сердец или щитов)
Получение урона:    Мигание красным + отбрасывание + неуязвимость 1 сек
Смерть:             HP = 0 → рестарт уровня с последнего чекпоинта
Аптечки:            Подбираются на уровне или выпадают из врагов
Чекпоинты:          Флаги/факелы, активируются при прохождении
```

- **Фигурки:** Лего-подобные, сборные из блоков
